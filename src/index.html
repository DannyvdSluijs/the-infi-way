De Infi Test #

Een vlugge lakmoesproef of je ruwweg The Infi Way volgt. Een soort Joel Test.

  1. [Proces] Zijn verantwoordelijkheden duidelijk belegd?
  2. [Proces] Zijn de PO, opdrachtgever en stakeholders voldoende betrokken?
  3. [Proces] Mag het team zelf passende technologie kiezen?
  4. [Code is de bron] Worden er automatische builds gemaakt na elke push?
  5. [Proces] Zijn er afspraken over het proces binnen het team?
  6. [Proces] Worden deze proces-afspraken periodiek geevalueerd?
  7. [Code is de bron] Zijn belangrijke keuzes gedocumenteerd?
  8. [Proces] Worden tickets adequaat gerefined voordat er aan gewerkt wordt?
  9. [Ontwikkelprincipes] Wordt code gereviewed voordat het gemergd wordt?
  10. [Code is de bron] Kun je automatisch uitrollen naar productie?
  11. [???] Worden secrets correct gemanaged?
  12. [???] Zijn er werkende backups waarvan je weet dat ze te restoren zijn?
  13. [???] Worden storingen preventief opgemerkt om downtime te minimaliseren?

Ontwikkelprincipes #

We werken op een moderne manier, gericht op kwaliteit. Goed is op de lange termijn ook het goedkoopst.

Hoe we dat graag doen:

  • Modern versiebeheer
  • Up to date tooling, frameworks, en libraries
  • Automation waar mogelijk
  • Sterke test coverage
  • Linting, code quality checks
  • Pair programming
  • Code reviews voor (vrijwel) alles

Denk aan bijvoorbeeld:

  • Git (flow op project afgestemd)
  • Unit tests, integration tests, end-to-end tests
  • Sonarqube, ESLint, etc.
  • SOLID (of DAMP), YAGNI, KISS
  • Domain-Driven Design

Hoe we dat graag doen:

  • Architectuur in de bron
  • Clone & run repositories
  • Continuous integration, continuous delivery
  • OTAP
  • Infrastructure as code

Proces en team #

Teams werken autonoom, maar kiezen vrijwel altijd een vorm van agile werken. Met Product Owner van de klant.

Hoe we dat graag doen:

  • Team bepaalt eigen proces
  • Retrospectives om het proces te blijven verbeteren
  • Product Owner vanuit de klant
  • Scrummaster (projectmanager of developer)
  • Samen een planning onderhouden
  • Alle teamleden helpen refinen
  • Minimaal 2 personen per team, splitsen vanaf 6 a 7
  • Collega's uit andere teams invliegen
  • Testers, designers, en overige expertise aanhaken

Denk aan bijvoorbeeld:

  • Scrum of Kanban
  • Trello, Jira, Clubhouse.io, YouTrack
